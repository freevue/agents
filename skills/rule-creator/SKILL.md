# Rule Creator

## Overview

이 기술은 프로젝트의 일관성을 유지하기 위해 필요한 규칙(Rule)과 컨벤션(Convention) 문서를 표준 규격에 맞춰 생성하도록 돕습니다. `skill-creator`의 Markdown 작성 원칙을 준수하여, 에이전트가 이해하기 쉽고 효율적인 규칙 시스템을 구축하는 것을 목표로 합니다.

## 핵심 원칙

1. **결정론적 명확성**: 규칙은 모호함 없이 에이전트가 즉각적으로 행동에 옮길 수 있도록 작성해야 합니다.
2. **최소주의 (Minimalism)**: 불필요한 서술을 배제하고 컨텍스트 창을 효율적으로 사용합니다.
3. **명령형 문체**: "해야 합니다"보다는 "하라", "실행", "금지" 등 직접적인 동작 지시어를 사용합니다.

## 규칙 생성 워크플로우

1. **목적 정의**: 생성하려는 규칙이 해결하려는 핵심 문제나 도메인을 정의합니다.
2. **템플릿 선택**: `references/rule-template.md`를 참고하여 표준 구조를 잡습니다.
3. **규칙 본문 작성**:
    - **Trigger**: 규칙이 활성화되어야 하는 조건(파일 확장자, 특정 키워드 등)을 명시합니다.
    - **Guidelines**: 구체적인 DOs와 DON'Ts를 나열합니다.
    - **Examples**: 올바른 예시와 잘못된 예시를 코드로 보여줍니다.
4. **검증 및 저장**: 작성된 규칙이 기존 컨벤션(`CODE_CONVENTIONS.md` 등)과 충돌하지 않는지 확인한 후 `.agent/rules/` 폴더에 저장합니다.
