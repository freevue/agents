# 에이전트 규칙 작성 가이드 (Rule Writing Guide)

## 문체 (Style)
- **명령형 사용**: "~하라", "~하지 마라", "사용 금지" 등 직접적인 동작 지시어를 사용합니다.
- **간결성**: 불필요한 수식어를 제거하고 핵심 행위(Action) 위주로 서술하여 에이전트의 컨텍스트 소모를 줄입니다.
- **일관성**: 동일한 개념이나 파일 경로 등은 전체 문서에서 동일한 용어로 지칭합니다.

## 포맷팅 (Formatting)
- **코드 블록**: 반드시 언어 식별자를 명시합니다 (예: \`\`\`typescript).
- **강조**: 중요한 키워드나 주의사항은 **볼드체**를 사용하거나 \`백틱\`으로 감쌉니다.
- **구조화**: 단계별 절차는 숫자 리스트(1, 2, 3), 일반 지침은 불렛 포인트(-)를 활용합니다.

## 에이전트 성능 최적화
- **트리거 명확화**: 에이전트가 이 규칙을 언제(어떤 파일 확장자, 어떤 작업 상황) 적용해야 하는지 명확히 명시합니다.
- **상호 참조**: 관련 있는 다른 규칙이나 컨벤션 문서가 있다면 링크를 제공합니다.
