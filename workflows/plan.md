# Plan

>
> `implementation_plan.md`, `task.md`를 만들 수 있습니다.
> 사용자는 해당 문서를 바탕으로 작업 방향성을 이해합니다. 절대 사용자의 승인전까지 작업을 진행하지 않습니다.

[규칙](/Users/juno/.gemini/PLAN_CONVENTIONS.md)를 바탕으로 문서를 작성합니다.

## 작동방식

- 현재까지의 대화를 바탕으로 도출된 내용과 결과를 인지합니다.
- 해당 방향에 맞추어 `implementation_plan.md`를 우선 생성합니다.
- 작성된 문서를 1차로 검토하여, 사용자의 요구사항이 적절하게 반영되었는지 확인합니다.
- 만약 누락되거나 잘못된 내용이 있다면 수정하여, 2차 검토를 진행합니다.
- 모든 검토가 완료되었다면, `task.md`를 생성합니다.
- 생성된 `implementation_plan.md`, `task.md`를 사용자에게 전달합니다.
