# Plan Conventions

> [!NOTE]
>
> 에이전트는 아래의 **핵심 인라인 가이드**를 기본 동작 원칙으로

## 작업계획서 양식

`implementation_plan.md`란 **작업계획서**를 뜻합니다. 아래의 내용을 반드시 포함해야 합니다.

### 작업제목

진행할 작업에 대한 목적을 알 수 있게 제목이 영역이 있어야 합니다.

### 작업내용

진행할 작업에 대해 이해한 사항을 서술형으로 간단히 작성해야합니다. 해당 섹션에 아래의 항목들이 반드시 기입되어야 합니다.

- 이번 작업으로 인해 수정되거나 추가되는 기능
- 이번 작업으로 인해 발생할 수 있는 사이드 이펙트
- 이번 작업을 통해 해결하고자 하는 문제점
- 이번 작업을 통해 참고해야할 자료

### 작업범위

작업을 진행할 파일들을 명시해야합니다. 아래는 작업 범위 작성에 대한 예시입니다.

```markdown
| File Link          | Task                    | For                       |
| ------------------ | ----------------------- | ------------------------- |
| [File1](File1.tsx) | [상세내용1](#상세내용1) | 사용자의 중 해당하는 부분 |
```

### 상세내용

[작업범위](#작업범위)에 들어갈 작업 내용을 파일별 구체적으로 정리한 영역입니다. 해당 영역은 각 수정항목마다 별도의 섹션으로 나누어 관리되어 [작업범위](#작업범위)에서 링크를 적용할 수 있어야 합니다.

```markdown
## 상세내용

### 상세내용1

{{작업을 진행하기 위한 목적과 사용자의 요청사항을 반드시 기입합니다.}}

\`\`\`
{{내용에 맞추어 수정될 사항이나 반영될 사항을 코드 블록으로 명시해야합니다.}}
{{블록의 경우 필요한 타입에 맞추어 사용하면 되지만, 기본값으로 `diff`형식을 사용해야합니다.}}
\`\`\`
```

### 검증방법

반영된 작업에 따라 검증할 수 있는 방법을 제시해야합니다. 필요에 따라 테스트 코드를 작성하여 출력합니다.

## 작업계획서 예시

```markdown
{{작업제목}}

{{작업내용}}

{{작업범위}}

{{상세내용}}

{{검증방법}}
```
